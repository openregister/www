Registers

A register is a datastore with high integrity, clear ownership and a standard API.

Registers have clear user needs and purpose and they also share certain characteristics.

Building services using registers

Registers present a standard Web API to access their data. They consistently use the same names for fields, allowing data to be easily linked across registers, which are potentially maintained by different organisations.

The following items help people understand the implications of incorporating the data held in a register into their services:

  • standards for presenting the provenance of the data
  • how to contact the team maintaining a register
  • the status of the register data (Alpha, Beta or Live)
  • the licensing terms for reuse

Ways to use registers

Product

A register can be installed on your own infrastructure using the register product.

Platform

Multiple registers may be hosted on a single platform, and made available as software as a service.

Standard

You can develop your own register software, and use the tests to confirm it meets the standard.

Registers and how they work

Sample Register

These are terms that describe the composition of a register.

Register

A register is an append-only lists of immutable entries. You can append entries to a register, but you cannot change the content of existing entries.

Entry

Each entry in a register is a set of field and value pairs. You can identify an entry using the serial number within a register, or using the hash of its contents.

Field

Each entry is comprised of a set of fields, each field is a name, used consistently across all registers. Each field name is recorded in the field register.

Value

The value of each field is constrained by the datatype, such as a string or datetime.

Primary Key Field

Each Register has a primary key which is a field which must appear in each and every entry. The primary key field has the same name as the name of the register.

Record

The set of all entries in a register that share the same primary key value is known as a record, and can be identified by the key value.

History

An entry is earlier than another if it has a lower serial number than the other in the register’s list. Hence ‘latest entry of a register’ or ‘latest entry of a record.’

Link

The definition of a field in the field register may indicate the value is to be treated as a reference to a record in another register. For example the string value FR in a country field may be treated a link to the entry with the key value FR in the country register, and the CURIE value school:1234567 indicates the entry in the school register with the key value 1234567.
