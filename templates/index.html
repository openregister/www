Characteristics of a register

A register consists of:

  • a single dataset conforming to a defined standard
  • a standard API for reading data
  • a standard API for adding to a register
  • automated tests for verifying the register-specific rules for data integrity
  • a digital certificate to sign changes to a register
  • a tamper-proof audit trail of changes made to a register

Aims of the register project

Over the course of an alpha and beta the aim is to demonstrate by way of working software, the following:

  • a standard for datatypes and field names for register entries
  • a software appliance that exhibits the charateristics of a register
  • a set of tools for independently verfiying the integrity of a register and the data held within it
